# Rua

Rua 是一门采用 Rust 风格语法、编译到可读 Lua 5.5 源码的静态类型脚本语言。它保留 `struct`、`enum`、`trait`、泛型、模式匹配、闭包和 iterator，但不实现所有权、借用检查或生命周期。

```bash
cargo build --release -p ruac -p rua-lsp --features lsp
target/release/ruac build app.rua
```

生成物面向 Lua 5.5。标准运行时集中在单个 [rua_std.lua](crates/rua-resources/resources/std/rua_std.lua)；bundle codegen 最多生成一次 `require("rua_std")`，modules codegen 依赖 Lua 的 `require` cache 共享同一个 runtime package。

## 完整示例：Rua 与 Lua

Rua 文件本身就是可执行 chunk，没有特殊入口函数。下面的源码覆盖顶层语句、结构体、
associated function、method、枚举、模式匹配、`Result`、`?`、Vec、HashMap、闭包、
iterator 和标准格式化函数：

```rua
struct Product {
    sku: String,
    price: i64,
}

impl Product {
    fn new(sku: String, price: i64) -> Product {
        Product { sku, price }
    }

    fn label(&self) -> String {
        format("{}: {}", self.sku, self.price)
    }
}

enum Checkout {
    Paid(Product),
    Rejected(String),
}

fn positive_quantity(quantity: i64) -> Result<i64, String> {
    if quantity > 0 {
        Ok(quantity)
    } else {
        Err("quantity must be positive")
    }
}

fn checkout(product: Product, quantity: i64) -> Result<Checkout, String> {
    let count = positive_quantity(quantity)?;
    let total = product.price * count;
    if total <= 10000 {
        Ok(Checkout::Paid(product))
    } else {
        Ok(Checkout::Rejected("limit exceeded"))
    }
}

fn describe(result: Result<Checkout, String>) -> String {
    match result {
        Result::Ok(Checkout::Paid(product)) => product.label(),
        Result::Ok(Checkout::Rejected(reason)) => reason,
        Result::Err(error) => error,
    }
}

let products = [
    Product::new("book", 4500),
    Product::new("keyboard", 12000),
];
let prices: HashMap<String, i64> = #{
    "book": 4500,
    "keyboard": 12000,
};
let total = products
    .iter()
    .map(|product| product.price)
    .fold(0, |sum, price| sum + price);
let result = checkout(Product::new("book", prices.get("book").expect("book price")), 2);

print("{}; catalog total={}", describe(result), total);
```

执行 `ruac build main.rua` 后得到以下 Lua 5.5 源码；这是当前 compiler 的实际输出：

```lua
-- Generated by ruac (Rua -> Lua 5.5). Do not edit by hand.
local rua_std = require("rua_std")
assert(rua_std.ABI_VERSION == 2, "incompatible rua_std ABI")
local fmt = rua_std.fmt
local map = rua_std.map
local result__runtime = rua_std.result
local vec = rua_std.vec
local tbcreate = table.create

---@class Product
---@field sku string
---@field price integer
local Product = tbcreate(0, 2)
---@class Checkout

---@return table
local function positive_quantity(quantity)
    if quantity > 0 then
        return result__runtime.ok(quantity)
    else
        return result__runtime.err("quantity must be positive")
    end
end

---@return table
local function checkout(product, quantity)
    local __t1 = positive_quantity(quantity)
    if not __t1[1] then return __t1 end
    local count = __t1[2]
    local total = product.price * count
    if total <= 10000 then
        return result__runtime.ok({ tag = "Paid", product })
    else
        return result__runtime.ok({ tag = "Rejected", "limit exceeded" })
    end
end

---@return string
local function describe(result)
    local __t2 = result[1]
    if __t2 and result[2].tag == "Paid" then
        local product = result[2][1]
        return Product.label(product)
    elseif __t2 and result[2].tag == "Rejected" then
        local reason = result[2][1]
        return reason
    else
        local error = result[2]
        return error
    end
end

---@return Product
function Product.new(sku, price)
    return {
        sku = sku,
        price = price,
    }
end

---@return string
function Product:label()
    return fmt.format("{}: {}", self.sku, self.price)
end

local products = vec.from_table({ [0] = Product.new("book", 4500), Product.new("keyboard", 12000), n = 2 })
---@type table
local __t3 = map.new(2)
__t3:insert("book", 4500)
__t3:insert("keyboard", 12000)
local prices = __t3
local total = 0
for __t5 = 0, products.n - 1 do
    local __t4 = products[__t5]
    __t4 = __t4.price
    total = total + __t4
end
local __t6 = prices:get("book")
if __t6 == nil then error("book price", 2) end
local result = checkout(Product.new("book", __t6), 2)

fmt.print("{}; catalog total={}", describe(result), total)
```

关键语义可以逐项对应：

| Rua | 生成的 Lua |
|---|---|
| `struct Product` | EmmyLua `Product` class 与 Lua table |
| `Product::new(...)` | `Product.new(...)` |
| `product.label()` | `Product.label(product)`；method 定义使用 `Product:label()` |
| `Ok(value)` / `Err(error)` | `result__runtime.ok(value)` / `result__runtime.err(error)` |
| `positive_quantity(...) ?` | 检查 `[1]` tag，错误时直接返回 tagged Result |
| `Checkout::Paid(product)` | `{ tag = "Paid", product }` |
| `[a, b]` | 0-based table 与显式 `n`，由 `vec.from_table` 包装 |
| `#{ key: value }` | `map.new(capacity)` 后按源码顺序插入 |
| `map(...).fold(...)` | 一个直接累加的 Lua `for` |
| `expect("message")` | 带调用方消息的 nil guard |

运行结果：

```text
book: 4500; catalog total=16500
```

### Binding、函数与表达式

分号结束 statement；block、`if`、`match` 和 `loop` 的尾表达式产生值：

```rua
fn clamp(value: i64, min: i64, max: i64) -> i64 {
    if value < min {
        return min;
    }
    if value > max { max } else { value }
}

let inferred = 42;
let typed: f64 = 3.14;
let mut count = 0;
count += 1;

let computed = {
    let base = 20;
    base + 22
};
```

### 控制流与模式

```rua
fn bucket(value: i64) -> String {
    match value {
        0 => "zero",
        1 | 2 => "small",
        3..=9 => "medium",
        n if n >= 10 => "large",
        _ => "negative",
    }
}

fn search(values: Vec<i64>, wanted: i64) -> Option<i64> {
    for index in 0..values.len() {
        if values[index] == wanted {
            return Some(index);
        }
    }
    None
}

let mut stack = [1, 2, 3];
while let Some(value) = stack.pop() {
    if value == 2 { continue; }
    print("{}", value);
}

fn ready() -> bool { true }

let answer = loop {
    if ready() { break 42; }
};
```

`loop` 允许 `break value`；同一 loop 的所有 break value 必须类型兼容。
`while` 与 `for` 是 statement，只接受裸 `break;`。

### Struct、enum、trait 与泛型

```rua
pub struct Point<T> {
    pub x: T,
    pub y: T,
}

pub enum Shape {
    Dot,
    Circle(f64),
    Rect { width: f64, height: f64 },
}

trait Named {
    fn name(&self) -> String;

    fn display(&self) -> String {
        format("[{}]", self.name())
    }
}

impl Named for Shape {
    fn name(&self) -> String {
        match self {
            Shape::Dot => "dot",
            Shape::Circle(_) => "circle",
            Shape::Rect { .. } => "rect",
        }
    }
}

fn label<T: Named>(value: T) -> String {
    value.display()
}

fn same<T>(value: T) -> T
where
    T: Named,
{
    value
}
```

inherent `impl Type` 定义 associated function 和 method；`impl Trait for Type` 验证
trait contract。`Add`、`Sub`、`Mul`、`Div`、`Rem` 等 operator trait 可为用户类型定义
二元操作符行为。

### Option、Result 与错误传播

```rua
fn load(path: String) -> Result<String, String> {
    if path == "" { Err("empty path") } else { Ok("config") }
}

fn load_length(path: String) -> Result<i64, String> {
    let text = load(path)?;
    Ok(text.len())
}

struct Profile {
    city: String,
}

let profile: Option<Profile> = Some(Profile { city: "Shanghai" });
let city = profile?.city ?? "unknown";
let required = profile.expect("profile is required");

let optional_value: Option<i64> = Some(42);
if let Some(value) = optional_value {
    print("{}", value);
}
```

`Option<T>` 和 `Result<T, E>` 提供 `map`、`unwrap`、`expect`、`unwrap_or` 与状态查询；
`?` 保留当前函数的 Option/Result 传播语义。

### Vec、HashMap、closure 与 iterator

```rua
let mut values = [1, 2, 3];
values.push(4);
let first = values[0];

let scores: HashMap<String, i64> = #{
    "alice": 10,
    "bob": 20,
};
let known = "alice" in scores;
let contains = 2 in values;
let substring = "ua" in "Rua";

let offset = 10;
let add_offset = |value: i64| -> i64 { value + offset };
let shifted = add_offset(first);
let selected = values
    .iter()
    .map(|value| value + offset)
    .filter(|value| value > 11)
    .collect::<Vec<i64>>();
let total = selected.iter().fold(0, |sum, value| sum + value);
```

iterator 支持 range/Vec source、`map`、`filter`、`filter_map`、`find`、`any`、`all`、
`count`、`fold`、`collect` 等 adapter/consumer；可证明的一次性 chain 直接生成融合循环。

### Module、extern、attribute 与 annotation

module identity 来自文件路径，源码通过 `use` 导入定义：

```rua
use math::add;
use math::trig::sin as sine;
use text::{join, split as words};

extern "lua" {
    fn host_log(message: String);
    fn host_format(template: String, ...) -> String;
}

#[cfg(feature = "http")]
pub fn serve() {}

#[targets(function)]
#[retention(runtime)]
pub annotation Route(method: String, path: String);

#[Route(method = "GET", path = "/health")]
pub fn health() -> String { "ok" }
```

`extern "lua"` 声明 ambient Lua global，不生成实现。attribute 的 project 配置、target、
retention、metadata 和 runtime registry 见
[Attribute 与条件编译](docs/rua-attributes.md)与 [Annotation](docs/rua-annotations.md)。

## 运行时语义

名为 `main` 的函数只是普通函数，不会被编译器自动调用；顶层语句按源码顺序执行。

`Result<T, E>` 是一等 tagged value，而不是 Lua multi-return。`Ok(value)` 和 `Err(error)` 在赋值、参数传递、字段、容器和闭包捕获后仍保留状态；`Ok(nil)` 与 `Err(nil)` 也可区分。`?`、模式匹配和普通返回使用同一表示。

```text
Ok(v)  -> { true,  v }
Err(e) -> { false, e }
```

第一个数组槽是 `is_ok`，第二个是 payload。boolean tag 与 payload 分离，所以
`Ok(nil)` 和 `Err(nil)` 仍可区分，同时避免为每个 Result 分配命名 hash 字段。

只有显式 extern/FFI adapter 可以把 tagged Result 转换成宿主约定。`Option<T>` 当前使用 `T | nil`，因此 `Some(nil)` 不属于可表达状态。
`Option` 与 `Result` 的 `map`、`unwrap`、`expect`、`unwrap_or` 和状态查询来自标准
`.ruai`；`expect(message)` 在缺失值或错误状态下携带调用方消息终止执行。

```rua
fn load(path: String) -> Result<String, String> {
    if path == "" { Err("empty path") } else { Ok("config") }
}

fn read_config() -> Result<String, String> {
    let config = load("app.rua")?;
    Ok(config)
}
```

常用脚本表达式保持静态类型，并生成显式 Lua 控制流：

```rua
struct Profile {
    city: String,
}

fn choose_city(profile: Option<Profile>, cities: Vec<String>) -> String {
    let mut attempts = 0;
    attempts += 1;

    let city = profile?.city ?? "unknown";
    let aliases: HashMap<String, String> = #{ "sh": "Shanghai" };
    let supported = city in cities && "sh" in aliases;

    loop {
        if supported { break city; }
        break "fallback";
    }
}
```

复合赋值的复杂左值只求值一次；`??` 仅在 `None` 时求值右侧，因而不会把
`Some(false)` 当成缺省；`?.` 的 method 参数也惰性求值。`in` 支持 `Vec`、
`HashMap` key、`String` 子串和 `Iter`/range，`#{...}` 构造有统一键值类型的
`HashMap`。`loop` 可用 `break value` 返回值，`while`/`for` 只允许裸 `break`。

其他核心映射：

| Rua | Lua 运行时表示 |
|---|---|
| `i64` / `f64` | Lua integer / number |
| `bool` | boolean |
| `String` | string |
| `Option<T>` | `T` / `nil` |
| `Result<T, E>` | `{ is_ok, payload }` tagged array table |
| `Vec<T>` | 0-based table，加 `n` 长度 |
| `HashMap<K, V>` | runtime map table |
| `struct` | table + class metatable |
| `enum` | tagged table |

`[1, 2, 3]` 是 `Vec<i64>` literal，空 `[]` 从 expected type 获得元素类型。格式化、
输出、panic 和断言使用标准 `.ruai` 声明中的普通函数，例如
`format("{}", value)`、`print("{}", value)` 和 `assert(condition)`，因此 compiler
与 LSP 共享相同的签名、文档和 definition。Rua expression grammar 包含普通函数调用
和 Vec literal；函数调用统一使用 `callee(...)`，Vec literal 使用独立的 expression node。

声明和成员可以使用 `#[cfg(...)]`、`#[cfg_attr(...)]` 进行 project 级条件编译。
`annotation` 声明提供带 target、参数和 retention 校验的结构化元数据；build metadata
可供外部工具扫描，runtime retention 只在实际使用时生成 registry。完整边界与示例见
[Attribute 与条件编译](docs/rua-attributes.md)和
[Annotation](docs/rua-annotations.md)。

## 模块

Rua 的模块身份直接来自文件路径，source grammar 中没有 `mod` item。以
`src/main.rua` 为入口时，`src` 是 source root：

```rua
use math::add;

let total = add(3, 4);
```

`src/math.rua` 和 `src/math/mod.rua` 都映射到 `math`，但不能同时存在；
`src/domain/order.rua` 映射到 `domain::order`，中间目录自动形成 namespace。
`.ruai` 使用同一规则。workspace `.rua` 优先于同路径 declaration 和外部库，
同一优先级的重复映射会报歧义，不会静默选取。

例如 `src/domain/product.rua`：

```rua
pub struct Product {
    pub sku: String,
    pub price: i64,
}

impl Product {
    pub fn new(sku: String, price: i64) -> Product {
        Product { sku, price }
    }

    pub fn label(&self) -> String {
        format("{}: {}", self.sku, self.price)
    }
}
```

入口 `src/main.rua`：

```rua
use domain::product::Product;

let product = Product::new("book", 4500);
print("{}", product.label());
```

执行 `ruac build src/main.rua --emit modules --out-dir dist` 后，入口
`dist/main.lua` 使用普通 Lua require：

```lua
-- Generated by ruac (Rua -> Lua 5.5 modules). Do not edit by hand.

local rua_std = require("rua_std")
assert(rua_std.ABI_VERSION == 2, "incompatible rua_std ABI")

local DomainProduct = require("domain.product")

local fmt = rua_std.fmt

---@class Main
local Main = {}

local product = DomainProduct.new("book", 4500)
fmt.print("{}", DomainProduct.label(product))

return Main
```

只有一个公开类型的 `dist/domain/product.lua` 直接返回 `Product`，不增加
`Product.Product` 层级：

```lua
-- Generated by ruac (Rua -> Lua 5.5 modules). Do not edit by hand.

local rua_std = require("rua_std")
assert(rua_std.ABI_VERSION == 2, "incompatible rua_std ABI")

local fmt = rua_std.fmt
local tbcreate = table.create

---@class domain.Product
---@field sku string
---@field price integer
local Product = tbcreate(0, 3)
Product.__index = Product

---@return domain.Product
function Product.new(sku, price)
    return setmetatable({ sku = sku, price = price }, Product)
end

---@return string
function Product:label()
    return fmt.format("{}: {}", self.sku, self.price)
end

return Product
```

`.ruai` 允许空 `{}` 作为 function/method declaration body，但任何非空实现或
顶层 executable statement 都以 `E0108` 拒绝。compiler 与 LSP 共享
`rua-project` 的路径映射；解析阶段为 module、定义、local、type、trait、variant
和 use site 分配稳定 identity，Lua codegen 只消费这些结果。

## Compiler

CLI：

```bash
ruac build src/main.rua                         # 写入 src/main.lua
ruac build src/main.rua -o dist/app.lua
ruac build src/main.rua --emit modules --out-dir dist \
  --lua-path dist --lua-path /path/to/rua_std
ruac check src/main.rua
ruac build src/main.rua --std-path ./path/to/standard-library
ruac build src/main.rua -c ./path/to/.ruarc.toml
ruac build src/main.rua --features http,trace --cfg runtime=moon
ruac metadata src/main.rua > dist/annotations.toml
```

默认 `--emit bundle` 把完整程序写入一个 Lua 文件。`--emit modules` 为每个有实体
`.rua` source 的 runtime module 生成一个文件：入口保留 root 文件名，
`domain::order` 写入 `domain/order.lua`。每个输出在顶部使用普通 Lua
`require("domain.order")` 加载依赖，依赖 local 使用完整逻辑路径的 PascalCase，
例如 `application.checkout` 对应 `ApplicationCheckout`。只有一个公开
`struct`/`enum` 的模块直接返回该类型表，例如 `domain/product.rua` 返回 `Product`，
调用方直接使用 `product.new()`；
多个公开类型或没有公开类型的模块返回模块表。模块表使用文件名末段的 PascalCase，
例如 `presentation/console.rua` 生成 `local Console = {}`。EmmyLua identity 使用完整
路径，例如 `---@class presentation.Console` 或 `---@class domain.Product`，避免不同目录
中的同名定义冲突。生成文件头按 Lua 搜索路径、`rua_std` ABI、module dependency 和
标准库 local alias 分组；类型、函数与最终 `return` 之间使用稳定空行。可用重复的
`--lua-path <dir>` 把搜索目录写入入口的 `package.path`：

纯目录 namespace 只参与编译期路径解析，不生成 `domain.lua` 之类的聚合文件，也不
产生额外 require；若需要 `require("domain")`，必须提供实体 `domain.rua`。

```bash
lua dist/main.lua
```

也可通过 `LUA_PATH` 设置运行环境，或在 `.ruarc.toml` 的
`runtime.lua_path` 中持久配置目录。modules 模式遵循 Lua `require` 的深度优先
初始化顺序；无法用普通 `require` 安全表示的循环模块依赖会在编译期报错，此类工程
可消除依赖环或继续使用 bundle。`--out-dir` 必须显式指定；编译器覆盖本次生成的
文件，但不删除目录中的其他内容。

存在 runtime annotation 时，modules artifact 包含一个聚合的
`rua_annotations.lua`。加载 registry 只注册 canonical locator，
`Annotations::load` 才按 module/path 惰性加载 target。

`ruac` 默认从输入文件目录向上查找最近的 `.ruarc.toml`。项目配置与 LSP
共享，字段使用 snake_case：

```toml
[workspace]
library = ["./types", "../host/moon.ruai"]

[[workspace.lua_library]]
root = "../moon_rs/lualib"

[workspace.library_mounts]
host = "../host/actual-name.ruai"

[runtime]
std_path = "./std"
lua_path = ["./dist", "/opt/rua/lib"]

[build]
features = ["http", "trace"]

[build.cfg]
runtime = "moon"
embedded = false
```

普通 Lua 库优先使用 `workspace.lua_library`。共置目录用 `root`；声明和
runtime 分离时写一次 root pair：

```toml
[[workspace.lua_library]]
declaration_root = "../moon_rs/ruai"
runtime_root = "../moon_rs/lualib"
```

声明按相对路径自动映射：`moon/http/client.ruai` 对应 Rua
`moon::http::client` 和 Lua `require("moon.http.client")`。每个文件型 `.ruai`
是独立 Lua module。compiler 与 LSP 都递归索引 declaration root；runtime root
自动加入生成入口的 `package.path`，因此无需逐模块配置或源码声明。

`workspace.library` 配置 declaration-only 搜索路径，`library_mounts` 配置显式逻辑
名称映射。命令行 `--library`、
`--library-mount name=path`、`--std-path` 和可重复使用的
`--lua-path dir` 可以覆盖或补充项目配置。`lua_path` 设置 bundle/modules 生成
入口的 Lua 运行时搜索目录，不参与 Rua/`.ruai` 的编译期 module 解析。

库 API：

```rust
let lua = ruac::compile_str("let value = 42;")?;
let lua = ruac::compile_path(path)?;
let lua = ruac::compile_path_with_std(path, std_root)?;
let lua = ruac::compile_project(&project_spec, &source_provider)?;
let artifact = ruac::compile_project_with_diagnostics(
    &project_spec,
    &source_provider,
)?;
let modules = ruac::compile_project_modules_artifact(&project_spec, &source_provider)?;
```

`compile_project_with_diagnostics` 是完整 host 集成入口，成功值包含 Lua 与 generated-to-Rua source map，失败值 `CompileFailure` 包含 diagnostic code、文件和 byte range。`compile_str`、`compile_path`、`compile_project` 与 artifact convenience API 同样返回结构化失败；只有 CLI 负责渲染展示文字。默认入口使用内嵌标准库；`compile_*_with_std` 和 `--std-path` 显式加载包含 `std.toml` 的目录。`compile_path_artifact` 为文件系统 host 保留 generated-to-Rua source map。`ProjectSpec` 提供稳定 `FileId`、逻辑路径、source root 和 library mount；`SourceProvider` 提供源码。

标准库的 `.ruai` 签名、文档、成员、Lua 包、导出子表、局部别名和可选 ABI 由 `std.toml` 统一描述。声明文件与 `rua_std.lua` 放在同一资源目录；`Vec`、`HashMap`、`Iter`、`String`、`Result`、格式化和整数运算是同一 Lua 包的独立导出。编译器只对 manifest 指定的 `Option` 与 `Result` language item 保留特殊表示。同名用户类型仍是普通 `struct`/`enum`，不会误入标准库 lowering。部署时只需提供一个 `rua_std.lua`。

编译器主链：

```text
rua-lex tokens
  -> strict parser / owned AST
  -> module collection + resolved HIR identities
  -> structural check + ID-keyed type facts
  -> backend layout
  -> structured Lua IR
  -> Lua printer
```

## IDE

Rua 使用两套 parser：

- `ruac` 使用 fail-fast strict parser，适合编译器和嵌入 host。
- `rua-syntax` 使用 error-tolerant Rowan parser，保留 trivia 和错误节点，适合编辑中的源码。

两者共享 `rua-lex`、`rua-core`、`rua-project`、grammar corpus 和 range conformance 测试，但不共享 AST、错误恢复或语义实现。`rua-analysis` 是独立的增量语义引擎，不调用 compiler AST/typeck 作为 production fallback。

`rua-lsp` 支持 hover、goto definition/implementation、completion、references、atomic rename、call/type hierarchy、inlay hints、diagnostics、semantic tokens、code actions、symbols、folding 和 formatting。函数、方法、trait method、extern 与 `.ruai` declaration 的文档来自统一 semantic record；`print`、`format`、`panic` 等标准函数直接从标准 `.ruai` 索引。enum variant 在构造、限定路径、alias 和 pattern 中使用同一 identity。

VS Code 设置：

- `rua.server.path`, `rua.server.args`, `rua.compiler.path`,
  `rua.compiler.args`, `rua.trace.server`。

library、module mount 和标准库路径只写入 workspace 根目录的 `.ruarc.toml`，
让 `ruac` 与 `rua-lsp` 使用同一份项目输入。VS Code 设置只控制 compiler、
language server 进程与协议 trace。Explorer 或编辑器中右键 `.rua` 文件可执行
**Rua: Build File**。

workspace/library 扫描支持 `.gitignore`、`.ignore` 和 `.ruaignore`，默认排除 `.git`、`target`、`node_modules`。
